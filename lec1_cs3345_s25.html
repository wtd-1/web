Introduction

Data as a general concept refers to the fact that some existing information or knowledge is represented or coded in some form suitable for better usage or processing. Data structures are the way data is organized, stored, and managed in a computer so that it can be accessed and modified efficiently. Put simply, data structures are any data representation and its associated operations; among these include placing a sorted list of integers in an array, for instance. This course will cover various data structures and algorithms to help you understand how to store and manipulate data effectively.

Most data structures have associated algorithms to perform certain operations on the data. Perhaps you might wish to search for a particular element, sort a list of numbers, insert elements, or balance (i.e., organize) the list. These operations are performed using algorithms which maintain the properties of their related data structure. In other words, data structures and algorithms go hand-in-hand. A match made in heaven!

Consider.
The LargestNumber Algorithm 


        Algorithm LargestNumber
        
        Input: A list of integers, L
        Output: The largest number in the list
        
        Procedure:
        1. Set max to the first element of L
        2. For each element x in L
        3.     If x is greater than max
        4.         Set max to x
        5. Return max
An algorithm is just a step-by-step procedure used to solve a problem. Here, the LargestNumber algorithm is fed a list of integers and is tasked with finding that greatest one in the list. To do that, LargestNumber starts with the first element of the list (we have to start somewhere!). After establishing a starting point, LargestNumber proceeds to search through the list using a loop. If the number it encounters at the current loop iteration is greater than the starting value, then the algorithm updates the starting value to the new, larger number. This process continues until the end of the list is reached, at which point the algorithm returns the largest number it found. Voilà! The LargestNumber algorithm.

Basic Rules of Exponents

Exponents are a fundamental concept in mathematics and computer science. Knowing them is key! Here are the basic rules you need to remember:

Zero Exponent Rule. Anything raised to the power of 0 is equal to 1. For example, \(x^0 = 1\), where \(x\) can be any integer. Typically, we assume \(x\) is positive, but this rule holds even for negative integers. For example, \(2^0 = 1\), \(3^0 = 1\), and so on.

Product Rule. When multiplying two powers with the same base, add their exponents. \[ x^a \cdot x^b = x^{a+b} \] Example. \(2^3 \cdot 2^4 = 2^{3+4} = 2^7\).

Quotient Rule. When dividing two powers with the same base, subtract the exponents. \[ \frac{x^a}{x^b} = x^{a-b}, \quad \text{for } x \neq 0 \] Example. \( \frac{5^6}{5^2} = 5^{6-2} = 5^4 \).

Power Rule. When raising a power to another power, multiply the exponents. \[ (x^a)^b = x^{a \cdot b} \] Example. \((3^2)^4 = 3^{2\cdot4} = 3^8\).

Negative Exponent Rule. A negative exponent means the reciprocal of the base with a positive exponent. \[ x^{-a} = \frac{1}{x^a}, \quad \text{for } x \neq 0 \] Example. \( 2^{-3} = \frac{1}{2^3} = \frac{1}{8} \).

Power of a Product Rule. When a product is raised to an exponent, distribute the exponent to each factor. \[ (xy)^a = x^a \cdot y^a \] Example. \( (2 \cdot 3)^4 = 2^4 \cdot 3^4 = 16 \cdot 81 \).

Power of a Quotient Rule. When a fraction is raised to an exponent, distribute the exponent to the numerator and denominator. \[ \left(\frac{x}{y}\right)^a = \frac{x^a}{y^a}, \quad \text{for } y \neq 0 \] Example. \( \left(\frac{4}{5}\right)^2 = \frac{4^2}{5^2} = \frac{16}{25} \).


Now that the rules of the game have been established, here are some of the applications!

Multiplication of Exponents

When the base is the same and the exponents are different, the exponents are added. For example: \(2^3 \cdot 2^2 = 8 \cdot 4 = 32\), which is equivalent to \(2^{3+2} = 2^5 = 32\). This is a fundamental rule of exponents!

Division of Exponents

When the base is the same and the exponents are different, the exponents are subtracted. For example: \( \frac{2^5}{2^2} = \frac{32}{4} = 8\), which is equivalent to \(2^{5-2} = 2^3 = 8\). This rule is often used in simplifying expressions.

Powers of Exponents

When an exponent is raised to another exponent, the exponents are multiplied. For example: \((2^3)^2 = 8^2 = 64\), which is equivalent to \(2^{3 \cdot 2} = 2^6 = 64\). It is important to note that this is multiplication of exponents, not addition.

Addition and Exponents

Exponents cannot be added directly during addition. For example: \(2^3 + 2^3 \neq 2^6\). Instead, \(2^3 + 2^3 = 2 \cdot 2^3 = 2^4\). Similarly, \(2^n + 2^n = 2 \cdot 2^n = 2^{n+1}\). This distinction is important for understanding exponential calculations!

Logarithms and Their Properties

In this class, unless specified otherwise, logarithms are assumed to have a base of two. For example, \(\log_2 64 = 6\), since \(2^6 = 64\). The following properties are useful:

  • Product Rule. \(\log_a (bc) = \log_a b + \log_a c\)
  • Quotient Rule. \(\log_a \left(\frac{b}{c}\right) = \log_a b - \log_a c\)
  • Power Rule. \(\log_a (b^n) = n \cdot \log_a b\)
  • Zero Rule. \(\log_a 1 = 0\), since \(a^0 = 1\) for any base \(a\).
  • Identity Rule. \(\log_a a = 1\), since \(a^1 = a\).
  • Inverse Rule. \(a^{\log_a x} = x\) and \(\log_a (a^x) = x\).
  • Change of Base Formula. \(\log_a b = \frac{\log_c b}{\log_c a}\), where \(c\) is any valid base.

Common Logarithms

The logarithm with base 10 is called the common logarithm and is written as: \[ \log_{10} x = \lg x \] For example, \(\lg 1000 = 3\), since \(10^3 = 1000\).

Natural Logarithms

The logarithm with base \(e\) (Euler's number, approximately 2.718) is called the natural logarithm and is written as: \[ \log_e x = \ln x \]

For example, \( \ln e^4 = 4 \), because the natural logarithm \( \ln x \) is the inverse of the exponential function \( e^x \). That is,

\[ \ln(e^x) = x \]

Applying this to \( e^4 \), we get:

\[ \ln e^4 = 4 \]

Logarithm and Exponent Relationship

Logarithms are the inverse of exponentiation. If \( a^x = b \), then \( \log_a b = x \). This fundamental relationship is used frequently in algebra and calculus. We use logarithms to solve for the exponent when the base and result are known. This is useful for when we get to more complex equations and functions.

Special Logarithmic Values

  • \(\log_a a = 1\), since \(a^1 = a\).
  • \(\log_a 1 = 0\), since \(a^0 = 1\).
  • \(\log_a a^x = x\), by the inverse property.
  • \(a^{\log_a x} = x\), by the inverse property.

Series

Summation series are used to calculate complexities in algorithms. For example:

  • Sum of the first \(n\) natural numbers: \(1 + 2 + 3 + \dots + n = \frac{n(n+1)}{2}\)
  • Sum of the squares: \(1^2 + 2^2 + 3^2 + \dots + n^2 = \frac{n(n+1)(2n+1)}{6}\)
  • Sum of the cubes: \(1^3 + 2^3 + 3^3 + \dots + n^3 = \left(\frac{n(n+1)}{2}\right)^2\)

Sum of the First n Integers

Let's compute the sum of the first n positive integers: \[ S = \sum_{i=1}^{n} i = 1 + 2 + 3 + \cdots + n. \] A clever trick to find a formula is to write the sum twice, once forwards and once backwards:

  • \( S = 1 + 2 + 3 + \cdots + n \)
  • \( S = n + (n - 1) + (n - 2) + \cdots + 1 \)

Adding these two together, each pair sums to \( (n+1) \), and since there are \( n \) pairs, we get: \[ 2S = n(n + 1). \] Divide both sides by 2, and we have our classic formula: \[ S = \frac{n(n + 1)}{2}. \] So, the sum of the first n integers grows as \( \Theta(n) \), meaning it scales linearly.

Sum of an Arithmetic Series

More generally, an arithmetic series looks like this: \[ S = \sum_{i=0}^{n} (a + i \cdot d) = a + (a + d) + (a + 2d) + \cdots + (a + nd). \] If we rewrite this sum a bit, we can split it into two parts:

  • One part is just \( (n+1)a \), which comes from adding \( a \) repeatedly.
  • The other part is the sum of the first \( n \) integers multiplied by \( d \), which we already know is \( \frac{n(n+1)}{2} \).

Putting it all together, we get: \[ S = \frac{(a + dn)(n + 1)}{2}. \] This formula is super useful when working with evenly spaced sequences!

When Things Get More Interesting...

Now, let's look at what happens when we sum powers of integers: \[ \sum_{i=0}^{n} i^c = \Theta(n^{c+1}) \quad \text{for any constant } c > -1. \] This tells us that as \( n \) grows, the sum grows at roughly the rate of \( n^{c+1} \). The higher the power, the faster the growth.

But what if \( c = -1 \)? That gives us the Harmonic series: \[ H_n = \sum_{i=1}^{n} \frac{1}{i} = \Theta(\log n). \] This means the sum grows logarithmically, much slower than polynomial growth. And if \( c < -1 \), the sum actually converges to a constant, meaning it stops growing altogether.

Geometric Series: The Exponential Cousin

Geometric series are even cleaner to work with. The simplest form is: \[ \sum_{i=0}^{n} 2^{i} = 2^{n+1} - 1. \] More generally, if we have some base \( A \), we get: \[ \sum_{i=0}^{n} A^i = \frac{A^{n+1} - 1}{A - 1}. \]

Here's a trick: if \( 0 < A < 1 \), then as \( n \to \infty \), the series approaches: \[ \frac{1}{1 - A}. \] This means if you keep adding up a fraction of the previous term, it eventually levels off at a fixed value.

Modular Arithmetic

Let's talk about modular arithmetic is like regular arithmetic but with a twist: everything "wraps around" at a certain number \( n \). Mathematically, we say that two numbers \( A \) and \( B \) are **congruent modulo \( n \)** if they leave the same remainder when divided by \( n \). This is written as: \[ A \equiv B \pmod{n} \] which is just a fancy way of saying that \( A - B \) is a multiple of \( n \): \[ A - B = k \times n \] for some integer \( k \). Example: Since both 81 and 61 leave a remainder of 1 when divided by 10, we can write: \[ 81 \equiv 61 \equiv 1 \pmod{10} \]

Modulo is Everywhere

  • Clock arithmetic: Ever noticed how 13:00 (1 PM) is just 1 modulo 12? That's modular arithmetic at work.
  • Cyclic patterns: Days of the week repeat every 7 days, meaning \( \text{Day} + 7 \equiv \text{Day} \pmod{7} \).
  • Cryptography: RSA encryption relies heavily on modular exponentiation to keep your messages safe.
  • Hashing & Computer Science: Hash tables and error detection (checksums) use mod to efficiently index and verify data.

Theorems

When working with modular arithmetic, we often assume \( n \) is a prime number. From that assumption, we get some beautiful theorems:

  • Theorem 1: Zero Product Property. If \( n \) is prime, and \( ab \equiv 0 \pmod{n} \), then at least one of \( a \) or \( b \) must be 0 modulo \( n \). In other words: \[ ab \equiv 0 \pmod{n} \quad \text{if and only if} \quad a \equiv 0 \pmod{n} \quad \text{or} \quad b \equiv 0 \pmod{n}. \] This is super useful in solving modular equations.
  • Theorem 2: Multiplicative Inverse. If \( n \) is prime, then for every \( 0 < a < n \), there exists a unique \( x \) such that: \[ ax \equiv 1 \pmod{n}. \] This means division exists in modular arithmetic, but instead of dividing, we use the multiplicative inverse of \( a \) mod \( n \). (This is a consequence of Bezout's Identity.)
  • Theorem 3: Quadratic Residues. The equation \( x^2 \equiv a \pmod{n} \) either has two solutions (mod \( n \)) or none at all, meaning \( a \) is either a quadratic residue or it isn't.

Bonus: Euler’s Theorem

If \( a \) and \( n \) are coprime, then: \[ a^{\phi(n)} \equiv 1 \pmod{n}. \] Here, \( \phi(n) \) is Euler's totient function, which counts how many numbers from 1 to \( n \) are relatively prime to \( n \). This is the backbone of RSA cryptography.

Bonus: Fermat’s Little Theorem

If \( p \) is a prime and \( a \) is any integer not divisible by \( p \), then: \[ a^{p-1} \equiv 1 \pmod{p}. \] This is a special case of Euler's theorem and is extremely useful for quickly finding inverses modulo a prime.

Recursion

Recursion is the idea that we have functions which call themselves. This can be a powerful tool in programming and mathematics. Here are some key points:

  • Terminating Rule. The object is defined explicitly.
  • Recursive Rule. The object is defined in terms of a simpler object, or itself.
  • Base Case. The simplest form of the object, which stops the recursion.
  • Recursive Case. The object in terms of a simpler object, which continues the recursion.
  • A function which is defined in terms of itself is said to be recursive.

For example, the factorial function is defined as:

\[ n! = n \times (n-1)! \]

with the base case \(0! = 1\). This means that the factorial of \(n\) is \(n\) times the factorial of \(n-1\), and the recursion stops when we reach 0, where the factorial is defined as 1.

And who said this had to just be math? You can code this too! 


        public static int factorial(int n) {
      
            if n == 0          // Establish the base case. 
                return 1;      
            else               // Make the recursive call. 
                return n * factorial(n - 1);
      
        }

Recursion Rules

Like everything, there are certain rules you need to follow:
  • Base Case. There must always be a base case you can solve without recursion.
  • Making Progress. For cases that are to be solved recursively, the recursive call must always be to a case that makes progress towards the base case. In other words, it has to be one of the preceding cases.
  • Design Rule. Assume all recursive calls work.
  • Compound Interest Rule. Never duplicate work by solving the same instance of a problem in separate recursive calls.

Recursive Functions

Recursive functions allow a function to call itself. This is a powerful tool in programming and mathematics, but it requires a **base case** to prevent infinite recursion. The professor emphasized this idea through two classic examples: factorials and Fibonacci numbers.

Factorial Function

The factorial of a number \( n! \) is defined recursively: \[ n! = n \times (n-1)! \] with the base case \( 0! = 1 \).

To compute \( 5! \), the function calls itself repeatedly:

\[ 5! = 5 \times 4! \quad \rightarrow \quad 4! = 4 \times 3! \quad \rightarrow \quad 3! = 3 \times 2! \quad \rightarrow \quad 2! = 2 \times 1! \]

The recursion stops at \( 1! = 1 \), then works its way back up, multiplying results:

\[ 5! = 5 \times 4 \times 3 \times 2 \times 1 = 120. \]

In code, a recursive factorial function looks like this: 


      public static int factorial(int n) {

          if n == 0        
              return 1;      
          else             
              return n * factorial(n - 1);

      }

Fibonacci Sequence

The Fibonacci sequence is another example of recursion: \[ F(n) = F(n-1) + F(n-2), \] with base cases \( F(0) = 0 \) and \( F(1) = 1 \).

To compute \( F(5) \), the function expands as follows:

\[ F(5) = F(4) + F(3) \quad \rightarrow \quad F(4) = F(3) + F(2) \quad \rightarrow \quad F(3) = F(2) + F(1) \] \[ F(2) = F(1) + F(0). \]

As the recursion proceeds, results are calculated and returned back up the chain.

Recursion vs. Iteration

While recursion provides a natural way to express problems, it can be inefficient. The Fibonacci sequence, for instance, results in redundant calculations—\( F(3) \) is computed multiple times.

To improve efficiency, memoization or dynamic programming stores intermediate results, avoiding repeated calculations. These are for CS 4349, Advanced Algorithms. In this course, just know of them--no need to implement them in CS 3345.

In some cases, an iterative approach is preferable, as it avoids excessive function calls and high memory usage. Understanding when to use recursion versus iteration is a key part of algorithm design.

Tail Recursion

Some recursive functions can be written in tail-recursive form, meaning the recursive call is the last operation performed. This allows some compilers to optimize recursion by reusing stack frames.

A tail-recursive factorial function: 


      public static int factorial(int n, int result) {
        
          if n == 0        
              return result;      
          else             
              return factorial(n - 1, n * result);

      }

Applications of Recursion

Recursion appears in many real-world applications, including:

  • Sorting algorithms like Merge Sort and Quick Sort.
  • Tree and graph traversal methods, such as Depth-First Search (DFS).
  • Mathematical problems like Towers of Hanoi and Greatest Common Divisor (GCD) using the Euclidean Algorithm.
  • Generating permutations and subsets in combinatorial problems.
